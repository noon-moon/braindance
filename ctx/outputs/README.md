# outputs

**Non-persisted outputs that don't belong in the vault.**

A scratch destination for work products an agent (or you) generates that
shouldn't be committed to the knowledge base — draft plans, generated reports,
one-off analyses, exports, intermediate artifacts. Everything in this directory
is **gitignored and ephemeral** (only this README is tracked).

- Write freely here instead of cluttering `ctx/vault/` with transient output.
- Nothing here is persisted or version-controlled, so don't rely on it lasting.
- If an output turns out to be worth keeping, **promote it into a vault note**
  in `ctx/vault/` (as a `memo`/`scope`) rather than leaving it here.

The mirror of `ctx/inputs/`: inputs come *in* for consideration, outputs go
*out* as disposable products. Neither is canonical — the vault is.
